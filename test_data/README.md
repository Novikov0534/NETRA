# NETRA — демо-наборы JSON

Формат каждого файла:

```json
{
  "nodes": [{ "id": "...", "label": "...", "status": "alive|dead" }],
  "links": [{ "source": "...", "target": "...", "load": 0.0-1.0 }]
}
```

Загружаются на вкладке **«Главная»** кнопкой «Загрузить JSON», после чего сайт
автоматически переходит на вкладку «Визуализация» и проверяет данные на целостность.
Если проверка находит проблемы — над графом появляется красный баннер со списком,
а на вкладке «Статистика» — блок «Проверка целостности данных» с полным перечнем.

## Корректные наборы

| Файл | Сценарий |
|---|---|
| `01_network_loaded.json` | Нагруженная сеть — высокая нагрузка на связях, 1 мёртвый узел |
| `02_network_dead_heavy.json` | Много неактивных узлов — почти половина сети недоступна |
| `03_network_sparse.json` | Разреженный граф — минимум связей, преимущественно линейная структура |
| `04_network_mixed.json` | Смешанная топология — разные статусы и уровни нагрузки |

## Наборы с намеренными ошибками

| Файл | Что сломано | Как увидеть на сайте |
|---|---|---|
| `05_error_missing_connection.json` | Связи ссылаются на узлы `sw-edge99` и `rtr-decommissioned01`, которых нет в `nodes` | В баннере ошибок: «соединение оборвано». На графе такие узлы отрисовываются как красные пунктирные ромбы с пометкой «⚠», а связь к ним — красным пунктиром |
| `06_error_duplicate_ids.json` | Узлы `sw-01` и `srv-02` объявлены дважды с разными label/status | Баннер покажет «Дублирующийся id узла… встречается 2 раза»; на графе дубликат отрисуется отдельным узлом с суффиксом `__dup2` |
| `07_error_broken_schema.json` | Узлы без `label`, без `status`, без `id`, пустой `id`; связь без поля `load`; статус `"offline"` вне допустимых значений | Баннер перечислит каждое отсутствующее поле; узел с неизвестным статусом закрасится серым |
| `08_error_invalid_load.json` | Нагрузка `1.8` (>100%), `-0.3` (отрицательная), `"high"` (строка вместо числа); самозамкнутая связь `sw-b → sw-b` | Баннер укажет на каждую связь; на графе такие связи подсвечиваются красным пунктиром |
| `09_error_isolated_nodes.json` | Узлы `orphan-sensor01`, `orphan-cam04`, `orphan-relay02` не участвуют ни в одной связи | Баннер: «изолирован — не имеет ни одной связи»; на графе такие узлы будут отображаться без рёбер |

Проверка не блокирует отрисовку — граф всё равно строится, а проблемные места
подсвечиваются, чтобы их можно было визуально найти и разобрать через панель деталей.
