# Формат JSON

Приложение принимает объект с двумя массивами: `nodes` и `links`.

```json
{
  "schemaVersion": "1.0",
  "datasetId": "monitoring:stand-01",
  "name": "Учебный стенд 01",
  "source": "monitoring",
  "generatedAt": "2026-07-24T12:00:00Z",
  "nodes": [
    { "id": "srv-01", "label": "srv-01", "status": "alive", "meta": { "type": "server" } },
    { "id": "srv-02", "label": "srv-02", "status": "dead" }
  ],
  "links": [
    { "source": "srv-01", "target": "srv-02", "load": 0.73 }
  ],
  "layout": {
    "positions": {
      "srv-01": { "x": 200, "y": 300 },
      "srv-02": { "x": 500, "y": 300 }
    }
  }
}
```

Обязательны только массивы `nodes` и `links`. Поля `schemaVersion`, `datasetId`, `name`, `source`, `generatedAt`, `meta` и объект `layout` необязательны.

## Узлы

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Уникальный идентификатор узла |
| `label` | string | Отображаемое имя узла |
| `status` | `"alive"` или `"dead"` | Состояние узла |
| `meta` | object | Необязательные данные источника: тип, хост, сервис и другие атрибуты |

## Связи

| Поле | Тип | Описание |
|---|---|---|
| `source` | string | `id` исходного узла |
| `target` | string | `id` целевого узла |
| `load` | number | Нагрузка связи от `0` до `1` |
| `meta` | object | Необязательные данные связи: протокол, порт и другие атрибуты |

Значение `load` влияет на цвет и толщину линии. Чем ближе значение к `1`, тем выше визуальная нагрузка.

## Фиксированное расположение

Необязательный объект `layout.positions` задаёт координаты узлов:

| Поле | Тип | Описание |
|---|---|---|
| ключ объекта | string | `id` узла |
| `x` | number | Горизонтальная координата |
| `y` | number | Вертикальная координата |

Координаты должны быть указаны для всех узлов. Если объект отсутствует или заполнен не полностью, NETRA строит одинаковую детерминированную раскладку по идентификаторам узлов и связям.

Состояния и метрики (`status`, `load`) можно обновлять без изменения координат: пока состав узлов и связей прежний, топология остаётся на месте. Это позволяет передавать в визуализатор данные из профайлера или сервиса мониторинга, не смешивая сетевые данные с состоянием интерфейса.

Для общей системы рекомендуется использовать стабильные строковые идентификаторы с пространством имён, например `host:server-01`, `container:web` и `service:postgres`.
